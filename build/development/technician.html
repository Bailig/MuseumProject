Museum Story Map Practice Guide

Overview: Three Main Steps

Step One: Data Preparation

Eleven comma-delimited tables in CSV format and a JSON format file will be created through the provided script tools.

The eleven CSV tables contain the data that are required to locate the collections on the map, and each CSV file contains a particular predefined group of collections.

The JSON file contains all the detailed information except location information. It will be used to show the description and image of the collections. The screenshot here shows one CSV and JSON example.

Step Two: Add Data to ArcGIS Online

Add the eleven CSV format files into your ArcGIS web map. Each CSV file will be a layer in the web map. The names of the layers and the order they are listed in your map are used by the story map tabs. The screenshot here shows eleven point layers added to the web map, which will each define one tab in the story map.

Step Three: Configure The Story Map Template

Configure the Shortlist template to use your web map by editing the index.html file. You can open this file in the editor of your choice. In the Config section of the file, edit the WEBMAP_ID variable to be the ID of your web map.

Put the JSON format file in the spesific location with spesific file name.

Data Preparation

1

If you are a computer programmer, you can prepare the data on your own without relying on the provided scripts. There are the utility and the required format of both CSV and JSON format files, which are required for creating the museum story map with the provided story map template.

Utility of CSV
The CSV format files should contain the data that are required to locate the collections on the map, and each CSV file should contain a particular predefined group of collections. It is because each CSV file will be a layer in the web map, and the names of the layers and the order they are listed in your map are used by the story map tabs.
Format of CSV
CSV stands for "comma-separated values". Its data fields should be separated, or delimited, by a comma. See How To Create A CSV File. By default, the valid fields names for creating the story map with the provided template are:
Valid fields names (case sensitive) Descriptions
LocationID Items sharing one location should have the same LocationID.
ArtifactID ArtifactID could be a foreign key.
Group Describes which tab the data belongs to.
Name The name of an item
Short_Desc A general description
Pic_URL Picture URL
Thum_URL Thumbnail URL
Latitude Latitude
Longitude Longitude
Point_Count The number of items sharing the location
You can find the sample CSV format files from the "sample/WebMapCSVs" folder.
Utility of JSON
The JSON file should contain all the detailed information except location information (Latitude and Longitude). It will be used to show the description and image of the items. When a Detail button from a popup window is clicked, the detailed information will be loaded from the JSON file into the browser by JavaScript.
Format of JSON
JSON stands for "JavaScript Object Notation". JSON is built on two structures. Fist, a collection of name/value pairs. Second, An ordered list of values. See JSON Tutorial. By default, the valid keys names for creating the story map with the provided template are:
Valid keys names (case sensitive) Descriptions
LocationID Items sharing one location should have the same LocationID.
ArtifactID ArtifactID could be a foreign key.
Tab_Name Describes which tab the data belongs to.
Name The name of an item
Short_Desc A general description
Pic_URL Picture URL
Thum_URL Thumbnail URL
ArtifactFinish Artifact finish
BeginDate Begin date
EndDate End date
Height Height
Width Width
Length Length
Manufacturer Manufacturer
NumberOfComponents The number of components
You can find the sample JSON format file from the "sample" folder.

2

If you are not a programmer, you can create the CSV and JSON format files with the provided scripts. However, you should at least know some basics of using a database to select and join tables.

  1. Get the geocoded data from the student team from computer programming field.
  2. Use your database to join the two tables, "artifacts.csv" and "category.csv", on key artifactID.
  3. Select all the records with the required fields that their group1, image, and thumbnail field must not be null, and at least either ManuProvince or ManuCity field is not null. Following is the SQL statement for this selection: 
    SELECT <required fields> 
    FROM <the joined table name> 
    WHERE image IS NOT NULL AND 
         group1 IS NOT NULL AND 
         (ManuProvince IS NOT NULL OR ManuCity IS NOT NULL);
    Following is the valid fields names (case sensitive): artifactID, ObjectName, GeneralDescription, Manufacturer, BeginDate, EndDate, NumberOfComponents, ArtifactFinish, group1, Length, Width, Height, image, thumbnail, lat, lng.
  4. Export the selected table to a comma-delimited CSV file. If you could not export it to comma-delimited CSV file, you could export it to a "|" delimited file and then run it through the provided Python script, "convertToCSV.py", to create the comma-delimited CSV file. You can find the Python script under the folder "script".
  5. Open the "script\src\DataPreparation.sln" file with Microsoft Visual Studio, an integrated development environment (IDE) from Microsoft.
  6. Navigate to "# region Set Variables". Assign the absolute input and output paths to the variables. For the descriptions of the variables, see the comments.
  7. Run the script by clicking on the Start button, as shown in the image.

The script should generate a JSON file and 12 CSV files, one master and 11 CSV files. You can find the sample CSV and JSON files from the "sample" folder.

Add Data to ArcGIS Online

3

In this step, we will add the 11 CSV format files into your ArcGIS web map. In your web map, zoom and pan to the area you want users to see when they first open your Story Map, and save your map. The title you save your map as is used as the title of your Story Map, and the summary is used as the subtitle.

  1. Log in to ArcGIS Online using a free, non-commercial ArcGIS public account or an ArcGIS subscription account, and create a new web map. Choose the basemap you want to use.
  2. Add the 11 CSV format files into your ArcGIS web map by clicking on "Add Layer From File". Each CSV file will be a layer in the web map. The names of the layers and the order they are listed in your map are used by the story map tabs.
  3. In your web map, zoom and pan to the area you want users to see when they first open your Story Map, and save your map. The title you save your map as is used as the title of your Story Map, and the summary is used as the subtitle.
  4. Share your web map. Normally you share your web map publicly so anyone can access your Story Map. But you also have the option, if you are using an ArcGIS subscription account, to share your map just within your organization if your Story Map is not intended for use by the general public.

Configure The Story Map Template

4

Configure the Story Map template to use your web map by editing the index.html file and copying the JSON format file created in step one into the specific location.

  1. Copy the contents of the Story Map template onto a publicly accessible website or web server. This can be a web server that you or your organization maintains, or it can be a folder on a shared or hosted web server that you use for your website.
  2. Open the "index.html" file in the editor of your choice. In the Config section of the file, edit the WEBMAP_ID variable to be the ID of your web map. The webmap ID is part of your web map URL.
  3. Copy the JSON format file created in step one into the spesific location "app\js\detailInfo.json" with default file name "detailInfo.json".
